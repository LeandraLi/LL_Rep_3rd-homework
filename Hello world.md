## **I. ОБРАБОТКА ТЕКСТА**

<br>

### **Выделение жирным**

**Жирный** __текст__ получается, если взять назначенный к выделению отрезок символов (текста, цифр и т.д.) в двойные звёздочки или двойные знаки нижнего подчёркивания с обеих сторон вплотную без пробелов: ** <текст> ** или __ <текст> __

### **Выделение курсивом**

*Наклонный* _текст_ получается, если необходимый отрезок символов взять в одинарные звёздочки или знаки нижнего подчёркивания с обеих сторон без пробелов между свёздочками и началом и концом текста: * <текст> * или _ <текст> _

### **Зачёркнутый текст**

~Зачёркнутый текст~ должен получаться при взятии необходимого отрезка в значки тильды с обеих сторон: ~<текст>~

### **Кодовые фрагменты строк**

Чтобы отметить фрагмент строки, содержащий код, необходимо окружить его обратными апострофами «`». При использовании кодовых фрагментов строк текст будет отображаться в виде моноширинного шрифта. В отличие от блоков кода, кодовый фрагмент позволяет поместить код внутрь обычного абзаца текста:

Используйте оператор `if`


***Все описанные выше методы можно комбинировать между собой.***

## **II. РАЗМЕТКА**

<br>

### **Новый параграф**

Создаётся путём пропуска одной и более строк.

<br>

### **Заголовки**

Заголовки разных утовней создаются путём добавления необходимого коичества знаков # в начале строки:

# Заголовок первого уровня
### Заголовок третьего уровня
###### Заголовок шестого уровня

<br>

### **Цитаты**

Цитаты создаются путём установки знака > в начале строки:

>Это пример цитаты,
>в которой перед каждой строкой
>ставится угловая скобка

Вложение цитаты в цитату:

> Первый уровень цитирования
>> Второй уровень цитирования
>>> Третий уровень цитирования
>
>Первый уровень цитирования

<br>

### **Списки**

Выделяются списки - с нумерацией и без.

***Упорядоченные (нумерованные):***

Для создания таких списков просто ставим цифру номера списка в начале строки с точкой + пробел:

1.	Проводник
2.	Полупроводник
      > *Цитата внутри элемента списка курсивом*
3.	Диэлектрик

***Неупорядоченные (ненумерованные):***

Для создания ненумерованных списков ставим один из трёх символов (*, - или +) в начале строки + пробел:

* Проводник
* Полупроводник
* Диэлектрик

- Проводник
- Полупроводник
- Диэлектрик

+ Проводник
+ Полупроводник
+ Диэлектрик

<br>

### **Блоки кода**

Блоки кода используются, когда необходимо процитировать исходный код программ или разметки.

Для создания блоков кода необходимо нажать Tab в начале строки, которая должна быть вожена.

* Проводник

        Блок кода 1

* Полупроводник

        Блок кода 2

* Диэлектрик

        Блок кода 3

<br>

### **Горизонтальные линии (разделители)**

Для визуального разделения двух частей текста используются горизонтальные линии.

Создаются они вводом в начале новой строки трех и более символов * или - (возможно использовать пробелы между ними):

***
---

## **III. КОМАНДЫ GIT**

<br>

### **Команда *git --version*** 

    Данная команда проверяет установленную на компьютере версию программы git и её наличие или отсутствие в принципе.

### **Команда *git init*** 

    Указываем (выбираем или создаём) папку, в которой git начнёт отслеживать изменения.

### **Команда *git status*** 

    Эта команда показывает текущее состояние git, есть ли в нём изменения, которые нужно сохранить (*commit*).

### **Команда *git add*** 

    Добавление содержимого рабочего каталога в индекс (*staging area*) для последующего коммита. После написания команды можно выбрать нужный файл, нажав Tab, чтобы не впечатывать его название вручную. 

### **Команда *git commit***

    Эта команда берёт все данные, собранные в индекс командой *git add*, и сохраняет их слепок во внутренней базе данных, а затем сдвигает указатель текущей ветки на этот слепок.

### **Команда *git log***

    Эта команда вызывает журнал изменений.

    Перед переключением версий файла в git эта команда используется, чтобы увидеть количество сохранений.

### **Команда *git checkout***

    Эта команда переключает между версиями файла в git.

    Команда git checkout master возвращает в тот коммит, в котором работаете.

### **Команда *git diff***

    Данная команда показывает разницу между текущим и сохранённым файлами.

    Перед сменой версии в git рекомендуется выполнить команду git log, чтобы увидеть количество сохранений.

### **Команда MKDIR** 

    используется для создания папки или подпапки через командную строку
